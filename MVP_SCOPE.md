# 最小 MVP 范围

本文档用于锁定效能引擎第一版最小可用范围，优先验证“飞书沟通 + 手工需求 + 任务分配 + 成果归档 + 进度统计”的闭环。

## 1. 飞书通信机制

MVP 必须实现：

- 飞书配置状态检测：确认 `App ID`、`App Secret`、机器人 Webhook、事件回调 Token 是否已配置。
- 飞书机器人通知：系统可向项目群或任务群发送文本通知。
- 飞书事件回调入口：先接收并记录飞书事件，为后续从群消息提取需求做准备。
- 飞书同步日志：所有发送、接收、授权类动作都要记录请求、响应、状态和错误。

暂不实现：

- 完整飞书 OAuth 登录。
- 自动读取群消息并提取需求。
- 飞书审批、日历、完整文档双向同步。

## 2. 需求收集

MVP 必须实现：

- 支持项目经理手工创建需求。
- 支持手工维护需求项，包括标题、描述、验收标准、优先级、预计工时。
- 支持需求确认和需求项确认。
- 保留 `source_type` 字段，当前默认 `manual`，后续可扩展为 `feishu_message`、`feishu_doc`、`ai_extract`。

后续增强：

- AI 接入飞书沟通群，自动识别客户需求、变更点和待确认问题。
- 将 AI 提取结果进入“待确认需求池”，由项目经理人工确认后入库。

## 3. 任务分配、目录权限与结果文件

MVP 必须实现：

- 支持把需求项生成任务。
- 支持把任务分配给员工。
- 分配时可以生成或登记任务成果目录。
- 系统记录“某任务对应哪个目录、授权给谁、授权状态是什么”。
- 员工完成后，把结果文件链接登记到任务下。

当前实现策略：

- 第一版先保存飞书目录 Token 或目录 URL。
- 目录授权先以 `mock_granted` 记录，表示后端流程已经预留，真实飞书云文档权限 API 后续替换。
- 结果文件先记录文件名、文件链接、飞书文件 Token、上传人和备注。

## 4. 任务进度与统计分析

MVP 必须实现：

- 任务状态：`todo`、`in_progress`、`blocked`、`pending_review`、`completed`。
- 任务进度百分比。
- 工时记录，并回写任务实际工时。
- 工作台统计：进行中项目、待处理任务、逾期任务、待适配需求、待审核报价、报价金额。
- 任务看板：按状态分组查看任务。

后续增强：

- 人员负载统计。
- 项目延期预测。
- 基于任务、工时、飞书消息的自动周报。
- AI 风险预警和资源调度建议。

## MVP 验收闭环

1. 管理员配置飞书机器人 Webhook。
2. 项目经理手工创建客户需求和需求项。
3. 项目经理把需求项生成任务并分配给员工。
4. 系统为任务登记成果目录，并记录目录授权状态。
5. 员工更新任务进度、记录工时、登记结果文件。
6. 管理者在工作台和任务看板查看进度与统计。
7. 系统通过飞书机器人发送任务分配或进度提醒。
